# A机构教育管理系统

这是一个面向教育机构的 AI 原生教研与学情管理系统。它不是单纯的学生信息表或成绩录入工具，而是把机构的课程标准、教师经验、学生表现证据、AI 评估和飞书预警串成一个闭环，用来沉淀机构自己的“教育大脑”。

系统的核心目标是：让机构能够快速建立课程评估标准，持续收集学生过程性证据，用 AI 生成可解释的六维度评价结果，并在异常发生时及时通知老师或管理者介入。

## 项目定位

本项目适合用于以下场景：

- 教培机构内部的学生、班级、考试和课程管理
- 素质类、体能类、艺术类课程的过程性评价
- 教研团队沉淀课程标准、考核规则和评分细则
- 老师批量录入成绩、上传作品或视频，生成学生画像
- 管理者查看机构整体教学状态、异常告警和 AI 评估结果
- 将机构经验通过 SaaS、API、知识包或私有化交付方式商业化

从产品形态上看，它更接近“AI 教研运营系统”，而不是传统教务系统。代码和资料中的设计思路强调：不要把所有课程标准提前硬编码，而是通过 RAG、模板、AI 生成和人工审核，让机构的标准可以动态演化。

## 目标用户

- 机构管理者：查看校区整体数据、教师班级概况、异常学生、AI 生成内容和评估结果。
- 授课老师：管理班级和学生，创建考试，导入成绩，上传学生作品、视频或客观题结果，审核 AI 评估。
- 教研负责人：维护课程、教案、考卷、考核标准、六边形维度和知识库内容。
- 运营/交付团队：将系统封装为 SaaS、API、标准包或白标私有化方案，服务其他教育机构。

## 核心业务闭环

系统围绕“标准 - 证据 - 评估 - 干预 - 沉淀”运转：

1. 课程与标准准备
   管理者或教研人员先配置课程、评定模板和六边形维度。系统默认包含素质评定和体能评定两套模板，也内置教案、考卷、考核标准生成框架。

2. 知识库沉淀
   教研资料、课程规则、评分标准、评测案例和历史结果进入 RAG 知识库。RAG 用本地 JSON 向量/词频存储做检索，AI 在回答问题、生成教案、生成考核标准和执行评定时都会引用这些资料。

3. AI 生成教研内容
   老板端的“教学生成中心”可以根据主题和框架生成教案、考卷、评分细则和考核标准。考卷会附带标准答案和主观题评分细则，考核标准会对齐六边形维度。

4. 学生证据采集
   老师可以录入考试成绩、批量导入学生/成绩、上传试卷图片、作品图片、体能视频，或录入客观题结果。体能视频可进入 MediaPipe 分析任务，书法/绘画作品会结合 OpenCV/OCR 与视觉模型生成指标。

5. AI 评定会话
   老师或管理者触发评定后，评定 Agent 会读取学生信息、课程轨迹、历史评定、姿态指标、表现证据和机构私有标准，生成草稿评定。系统要求关键维度有机构私有标准，否则会阻止评定，避免 AI 凭空打分。

6. 人工审核与校准
   AI 输出先是草稿。老师可以审核、修改分数、补充理由或驳回。老师手动调分会作为“人工校准案例”回写知识库，用于后续标准优化。

7. 最终归档与预警
   审核通过后可生效归档，评定结果会写入 RAG。若平均分过低、低分维度过多或证据不足，系统会生成异常告警，并通过飞书 Webhook 推送给对应老师和全局管理者。

8. 标准持续优化
   系统支持基于半年度评定样本生成标准草稿，供管理者审核、发布新版本标准，形成“数据反哺教研”的长期机制。

## 角色工作台

### 登录入口

首页是登录页，支持老板和老师角色。演示数据初始化后会创建：

- 管理员：`boss / boss123`
- 张老师：`teacher1 / teacher123`
- 李老师：`teacher2 / teacher123`

生产环境请务必更换默认账号密码和 `JWT_SECRET`。

### 老板端

老板端主要面向机构管理和教研管理：

- 总览仪表盘：查看学生、老师、考试、异常等关键指标。
- 教师班级概况：查看老师和班级维度的数据。
- 异常告警：集中查看学生异常、班级异常、重复错题和综合评定风险。
- 教学生成中心：管理生成框架，生成教案、考卷、考核标准，上传材料进行 AI 评估。
- AI 知识库：上传、查看、检索机构资料和评定标准。

### 老师端

老师端围绕日常教学执行：

- 班级管理：创建班级，配置班级飞书 Webhook。
- 学生管理：新增学生、批量导入学生、查看学生详情和画像。
- 考试管理：创建考试、导入成绩、记录错题。
- 多模态评估：上传体能视频、书法/绘画作品、客观题结果，并把指标写入评定证据。
- AI 评定审核：触发学生评定、查看维度得分、人工修改、审核生效。
- 异常告警：查看和处理本班学生风险。

## 评估维度

系统默认有两类六边形评定模板：

素质评定：

- 专注力
- 表达力
- 审美力
- 创造力
- 协作力
- 自主学习力

体能评定：

- 柔韧性
- 协调性
- 核心力量
- 耐力
- 爆发力
- 动作规范度

这些维度不是固定死的业务上限。后续可以通过数据库模板、RAG 标准文档和生成框架扩展新的课程、维度和评估规则。

## 主要功能模块

- 认证与权限：基于 JWT，区分 `boss` 和 `teacher`，老师只能访问自己班级学生。
- 学生/班级/考试管理：支持班级、学生、考试、成绩、错题和课程关联。
- 批量导入：支持 Excel/CSV 导入学生和成绩，并提供模板下载。
- RAG 知识库：支持手动上传资料、自动积累考试数据、告警数据、评测案例和最终评定结果。
- AI 内容生成：生成教案、考卷、评分细则和考核标准。
- 多模态评估：支持试卷图片、作品图片、视频/姿态指标和客观题判分。
- 评定 Agent：基于机构标准、学生历史、证据和姿态指标生成可解释评分。
- 人工审核：老师可修改 AI 分数，修改结果会沉淀为校准案例。
- 飞书通知：支持全局 Webhook 和班级/老师专属 Webhook。
- 标准版本迭代：支持生成半年度标准草稿、发布新标准、回滚标准。

## 数据与知识资产

项目中真正有业务价值的资产主要是：

- 机构私有评定标准
- 课程教案和考核规则
- 历史学生表现证据
- 教师审核和人工校准记录
- `data/vector_store.json` 中沉淀的 RAG 资料
- `data/education.db` 中的课程、模板、维度、评定和告警数据

这些数据默认不会上传到 GitHub。公开仓库只应保留代码、示例环境变量和说明文档。

## 商业化方向

资料中提到的商业化路径可以归纳为四类：

- AI-SaaS 智能教研托管平台：给其他机构开通账号，提供学生管理、AI 评估和标准库能力。
- 数据智能 API：对外提供评估、分析、建议生成接口，接入其他机构已有系统。
- 数字化教育 IP 标准包：把单门课程的评估标准、维度、评分阈值和案例样本打包销售。
- 私有化白标交付：为机构部署独立系统，替换品牌、域名和视觉，数据存放在客户自己的服务器。

当前代码更偏单机构 MVP。若要做 SaaS，需要继续补多租户、机构隔离、计费、权限边界和审计能力。

## 已知部署风险

- `node:sqlite` 在当前 Node 版本中仍可能出现实验性提示，部署时请固定 Node 22+ 并充分测试。
- `xlsx` 依赖存在 npm audit 告警且官方自动修复不可用。如果系统开放给外部用户上传 Excel，建议增加文件大小、字段、MIME、行数和内容校验，或评估替换导入库。
- 完整清除 Next.js 审计告警需要升级到 Next 15/16 大版本，属于迁移工作，不建议在未测试的情况下直接 `npm audit fix --force`。
